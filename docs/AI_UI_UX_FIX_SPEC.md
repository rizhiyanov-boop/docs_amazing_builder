# TЗ для ИИ-кодинга: UI/UX cleanup в `doc-builder`

## Контекст
- Приложение уже использует единый orchestration-слой в `src/App.tsx` и токены/адаптивность в `src/App.css`.
- Нужно исправить UX-проблемы без смены стека и без крупной перестройки бизнес-логики.
- Следуем текущим паттернам проекта: локальные helper-функции, существующие состояния `tab`, `isSidebarHidden`, topbar-кнопки, `panel`/`tabs`/`sidebar` CSS-классы.

## Цели
1. Сделать экспортный поток предсказуемым.
2. Убрать путаницу между сущностями `проект` и `метод`.
3. Улучшить мобильную навигацию без переписывания layout с нуля.
4. Повысить discoverability действий редактирования.
5. Актуализировать архитектурную документацию под реальное состояние проекта.

## Обязательные изменения

### 1. Export / preview flow
- Кнопка JSON в topbar остается прямым скачиванием.
- Кнопки HTML и Wiki в topbar больше не должны притворяться скачиванием.
- HTML и Wiki должны открывать соответствующий preview через состояние `tab`.
- Для Wiki нужно восстановить реальный переход в `tab === 'wiki'`.
- Внутри preview должны быть явные действия:
  - HTML: скачать файл.
  - Wiki: копировать текст и скачать текстовый файл.
- В рабочей области должны появиться явные вкладки `Редактор / HTML / Wiki`, используя существующий паттерн `.tabs` / `.tab`.

### 2. Терминология
- В multi-method UI оставить `проект` только для workspace/project-level сущности.
- Для элементов списка методов и их действий использовать `метод`.
- Исправить placeholder, warning, aria-label и title там, где сейчас метод назван проектом.

### 3. Мобильная навигация
- На compact viewport sidebar должен работать как drawer поверх рабочей области.
- Нужен backdrop для закрытия drawer.
- После выбора метода или секции на compact viewport drawer должен закрываться автоматически.
- Не ломать существующий desktop-паттерн sticky sidebar + resize handle.

### 4. Discoverability редактирования
- Не убирать текущую логику inline editing.
- Сделать action-кнопки редактирования заметными по умолчанию, сохранив усиление на hover/focus.
- Избегать сложных новых состояний и не переписывать все таблицы.

### 5. Документация
- Обновить `ARCHITECTURE.md` так, чтобы он не описывал проект как frontend-only SPA без используемого server/API слоя.

## Ограничения
- Не трогать security-аспекты.
- Не выносить большой рефакторинг по компонентам в этом проходе.
- Стараться менять только `src/App.tsx`, `src/App.css`, `ARCHITECTURE.md` и добавить отдельный spec-файл.

## Критерии приемки
- Из topbar можно открыть HTML preview и Wiki preview.
- Wiki preview достижим без обходных путей.
- В Wiki preview доступны copy и download.
- В интерфейсе метода больше нет подписей вида `Добавить проект`/`Удалить проект` рядом с методами.
- На мобильной ширине sidebar работает как overlay drawer.
- Кнопки быстрых edit-actions видны без hover-only зависимости.
- Сборка проекта проходит.
